لوحة مستودعات Nawah Token

روابط وتوصيات النشر النهائية لكل مشروع

1. nawahpi

الفرع للنشر: main

مجلد النشر: /root

الرابط المتوقع: https://username.github.io/nawahpi/

  • تحقق من وجود index.html وجميع ملفات CSS وJS والصور.
  • راجع الصور وملفات JS قبل النشر النهائي.

2. pi

الفرع للنشر: main

مجلد النشر: /root

الرابط المتوقع: https://username.github.io/pi/

  • تحقق من وجود index.html وجميع الملفات.
  • إذا استخدمت ملف .env، تأكد من عدم رفع البيانات الحساسة.

3. nawah-ui

الفرع للنشر: main

مجلد النشر: /root

الرابط المتوقع: https://username.github.io/nawah-ui/

  • تأكد من بناء واجهة React أولاً (npm run build).
  • تحقق من أن الملفات الناتجة index.html, CSS, JS موجودة في المجلد الصحيح.

4. password-vault-app

الفرع للنشر: main

مجلد النشر: /root

الرابط المتوقع: https://username.github.io/password-vault-app/

  • تحقق من الملفات التنفيذية وواجهة المستخدم.
  • تأكد من عمل ملفات JS وCSS بشكل كامل قبل النشر.

خطوات النشر النهائية لكل مستودع

  • تأكد من وجود الملفات الأساسية في الفرع الصحيح.
  • اذهب إلى Settings > Pages لكل مستودع:
    • Build and deployment: Deploy from a branch
    • اختر الفرع والمجلد (/root)
    • اضغط Save
  • انتظر دقائق قليلة وافتح الرابط المتوقع للتأكد من عمل الموقع.
  • (اختياري) ربط نطاق مخصص في Settings > Pages > Custom domain إذا أردت استخدام nawah.org.
